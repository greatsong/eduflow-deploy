# 관리자 초기 세팅 가이드

에듀플로를 [셀프 호스팅으로 배포](./DEPLOY.md)한 직후 관리자 한 분이 해야 할 일을 정리했어요. 이 문서의 단계를 한 번 끝내고 나면 선생님들께 도메인을 공유하실 수 있어요.

> 이 문서는 **운영자/관리자용**입니다. 선생님(사용자)이 보실 문서는 [USER_GUIDE.md](./USER_GUIDE.md)입니다.

---

## 당신이 관리자인지 확인

배포 시 `ADMIN_EMAILS`에 넣었던 Google 계정으로 로그인했다면 자동으로 관리자 권한이 부여돼요.

확인 방법:
1. 배포 도메인에 Google로 로그인
2. EntryForm(지원 정보) 작성 완료
3. **좌측 사이드바에 "관리자" 메뉴**가 보이는지 확인

사이드바에 관리자 메뉴가 없다면 `ADMIN_EMAILS` 설정을 다시 점검하세요:
```bash
fly secrets list | grep ADMIN_EMAILS
# 없거나 오타면
fly secrets set ADMIN_EMAILS="you@yourschool.kr"
# 이후 자동 재배포 대기
```

## 운영 유형부터 정하기

사용 맥락에 따라 권장 설정이 다릅니다. 먼저 본인 상황에 가까운 걸 고르세요.

### 유형 A. 학교·단체 내부용 (10~100명)

- **알려진 선생님들만** 쓰는 환경. 학교 구성원, 동아리, 소속 교사 모임 등
- **권장**: `registrationMode: approval` · `apiMode: server` · AI 키는 **관리자가 제공**
- 이유: 외부인 가입 차단, 비용 관리 용이, 선생님은 키 없이 즉시 사용

### 유형 B. 공개 커뮤니티 (누구나 가입)

- 온라인 연수, 공개 워크숍, 오픈 커뮤니티
- **권장**: `registrationMode: open` · `apiMode: user` · 관리자 키 **비공개**
- 이유: 승인 부담 없이 자유 가입, 비용은 각자 자기 키로 부담

### 유형 C. 본인 혼자/소수 신뢰 그룹

- 개인 운영자 + 가족/친한 동료 몇 명
- **권장**: `registrationMode: approval` · `apiMode: server` · 관리자 키 **공개**
- 이유: 신뢰 그룹에 한해 편의 극대화 (키 공유 필요 없음)

> 혼자만 쓰실 거면 [LOCAL.md](./LOCAL.md) 쪽이 더 맞으실 수 있어요.

## 관리자 대시보드 둘러보기

사이드바 **관리자** 메뉴로 들어가면 5개 탭이 있어요 (Admin.jsx).

| 탭 | 하는 일 |
|---|---|
| **사용자** | 가입자 목록, 상태 변경(active/inactive/pending), 지원 정보 조회 |
| **프로젝트** | 전체 프로젝트 현황, 삭제(에듀플로 + 포트폴리오 + GitHub 리포까지) |
| **설정** | 가입 모드, API 키 정책, 허용 모델 |
| **통계** | 사용자·프로젝트·챕터 수, 일별/주별 추이 |
| **토큰 사용량** | 프로바이더별 입력/출력 토큰, 누적 비용 |

---

## 1단계: 설정 탭

여기서 서비스 성격이 결정됩니다. 한 번 정하고 필요할 때만 바꾸시면 돼요.

### 1-1. 가입 모드 (`registrationMode`)

| 값 | 의미 |
|---|---|
| `open` | 신규 가입자가 **즉시 active**. 승인 불필요 |
| `approval` | 신규 가입자가 **pending**으로 대기. 관리자가 승인해야 사용 가능 |

학교/단체 내부용이라면 `approval` 권장. 외부인이 가입해도 pending에서 차단되니 안전해요.

### 1-2. API 키 정책 (`apiMode`)

| 값 | 동작 |
|---|---|
| `server` | 관리자가 등록한 키를 서버가 제공. 사용자는 키 입력 없이 바로 사용 |
| `user` | 사용자가 각자 자기 키를 브라우저에 입력해야 함 |

`server` 모드여도 사용자가 자기 키를 입력하면 그 키가 우선 사용됩니다. 우선순위는 이렇습니다:

```
사용자 입력 키 (헤더)  >  관리자 설정 키  >  환경변수(fly secrets)
```

### 1-3. 관리자 API 키 관리

설정 탭에서 프로바이더별로 키를 넣고, 각 키에 대해 이 두 가지를 정할 수 있어요.

- **공개 여부**: 공개하면 모든 사용자가 그 프로바이더의 모델을 쓸 수 있음. 비공개면 관리자 전용(테스트용)
- **프로바이더**: Anthropic, OpenAI, Google, Upstage

**일반적인 세팅**: Anthropic 키 하나만 **공개**, 나머지는 필요할 때 비공개로 추가해서 테스트.

### 1-4. 허용 모델 (`allowedModels`)

기본은 모든 등록 모델 허용이에요. 비용을 아끼고 싶으면 Haiku·Sonnet만 체크해서 Opus를 막을 수 있어요.

```
체크하면 보이는 것:
□ claude-opus-4-7        (비싸요, 중요한 교재용)
☑ claude-sonnet-4-6      (기본 권장)
☑ claude-haiku-4-5       (빠르고 저렴)
□ gpt-5-4                (비싸요)
☑ gpt-4-1                
☑ gemini-3-1
☑ solar-pro
```

## 2단계: 사용자 탭 — 첫 승인

`approval` 모드를 고르셨다면, 선생님이 가입 신청하실 때마다 **pending** 상태로 대기해요. 관리자가 승인해주기 전까지는 워크플로우에 진입하지 못합니다.

### 승인 절차
1. 사용자 탭 → **pending** 상태 필터
2. 이름·소속·지원 동기 확인
3. **active로 변경** 클릭

### 사용자 상태 의미
| 상태 | 의미 |
|---|---|
| `active` | 정상 사용 중 |
| `pending` | 승인 대기 (approval 모드 신규 가입) |
| `inactive` | 관리자가 비활성화 — 재로그인 시 차단됨 |

### 관리자 추가하기
`ADMIN_EMAILS`에 이메일을 추가하고 재배포하면 됩니다.
```bash
fly secrets set ADMIN_EMAILS="you@school.kr,another@school.kr"
```

## 운영 작업 모음

### 프로젝트 강제 삭제
- 프로젝트 탭 → 삭제 버튼 → 에듀플로 프로젝트 + 포트폴리오 항목 + (연결돼 있다면) GitHub 리포까지 함께 제거
- **복구 불가** — 사용자에게 먼저 확인하는 걸 권장

### 비용 모니터링
- **토큰 사용량 탭**: 프로바이더별 입력·출력 토큰 · 누적 USD
- 사용자별 집계도 나와요 — 특정 사용자가 급증하면 안내 가능

### 가입 정책 바꾸기
- `approval` → `open`: 대기 중이던 pending 사용자들은 **자동으로 active되지 않아요**. 사용자 탭에서 일괄 승인 필요
- `open` → `approval`: 이후 가입자부터 pending. 기존 active는 그대로

### 모델 추가·차단
- 설정 탭의 허용 모델 체크박스로 즉시 반영
- 차단된 모델을 이미 선택한 프로젝트는 기본 모델(`claude-sonnet-4-6`)로 폴백

### 볼륨 백업
```bash
fly ssh console
tar czf /tmp/backup-$(date +%F).tar.gz /data
exit
fly ssh sftp get /tmp/backup-$(date +%F).tar.gz ./
```

주기적으로(주 1회 이상) 받아두시면 안전해요.

## 문제 해결

| 증상 | 원인·해결 |
|---|---|
| 관리자 메뉴가 안 보여요 | 로그인 이메일이 `ADMIN_EMAILS`에 없음. 오타 확인 후 `fly secrets set`으로 재설정 |
| 승인했는데 사용자가 pending으로 보여요 | 사용자가 로그아웃 → 재로그인 필요 (JWT 재발급) |
| 선생님이 "API 키가 필요하다"는 메시지를 봐요 | `apiMode: server`인데 관리자 키가 **공개**로 설정 안 됨. 설정 탭에서 해당 프로바이더 키를 공개로 전환 |
| 토큰 사용량이 급증해요 | 토큰 탭에서 사용자별 집계 확인. 필요시 해당 사용자만 `inactive`로 일시 정지 또는 허용 모델 제한 |
| 특정 모델이 목록에 없어요 | `model_config.json`에 등록돼 있고 허용 모델 체크박스에 체크돼 있는지 확인 |
| 전체 로그를 보고 싶어요 | `fly logs` — 실시간. `fly logs > eduflow.log`로 저장 가능 |
