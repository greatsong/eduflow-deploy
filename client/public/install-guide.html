Installation Guide

선생님을 위한
에듀플로 설치 가이드

Windows 노트북에 10분이면 설치할 수 있어요. 명령어는 복사해서 붙여넣기만 하면 됩니다.

v0.5.1 · LOCAL_MODE Windows 10 / 11 약 10분 소요
Part 1 · Setup

처음부터 설치하기

Git → Node.js → 에듀플로 다운로드 → API 키 설정 → 실행. 다섯 단계만 지나면 바로 쓰실 수 있어요.

전체 과정
#할 일소요 시간
1Git 설치3분
2Node.js 설치3분
3에듀플로 다운로드3분
4API 키 설정1분
5실행30초
1

Git 설치

Git은 에듀플로 소스 코드를 내려받는 도구예요.

  1. git-scm.com/download/win 접속 → 자동으로 다운로드가 시작됩니다
  2. 받은 .exe 파일을 더블클릭
  3. 설치창이 뜨면 Next만 계속 누르다가 마지막에 Install → Finish

설치 확인: Windows 키 → cmd 입력 → 명령 프롬프트를 열고 아래를 입력하세요.

명령 프롬프트
git --version
# git version 2.xx.x 가 나오면 OK
에러가 나면
명령 프롬프트를 닫고 새로 열어서 다시 시도하세요. 그래도 안 되면 컴퓨터를 재시작한 뒤 재시도.
2

Node.js 설치

Node.js는 에듀플로를 실행하는 엔진이에요.

  1. nodejs.org 접속 → 초록색 LTS 버튼 클릭
  2. 받은 .msi 파일을 더블클릭
  3. Next 반복 → 동의 체크 → Install → Finish

설치 확인: 새로 연 명령 프롬프트에서 (기존 창은 닫고 열어야 해요)

명령 프롬프트 (새로 열기)
node --version
# v22.xx.x
npm --version
# 10.x.x
3

에듀플로 다운로드

아래 네 줄을 순서대로 복사해서 명령 프롬프트에 붙여넣으세요.

명령 프롬프트
cd %USERPROFILE%\Desktop
git clone https://github.com/greatsong/eduflow-deploy.git
cd eduflow-deploy
npm install

npm install은 1~3분 걸려요. 노란색 warn은 무시해도 되고, 빨간색 error만 아니면 성공입니다.

OneDrive 트랩
"지정된 경로를 찾을 수 없습니다" 가 뜨면 바탕화면이 OneDrive에 연결되어 있어요. 아래로 바꿔 시도하세요. 
cd "%USERPROFILE%\OneDrive\Desktop"
# 또는 한국어 경로
cd "%USERPROFILE%\OneDrive\바탕 화면"
4

API 키 설정

에듀플로 폴더 안 .env.example을 복사해 .env를 만들고 메모장으로 엽니다.

명령 프롬프트 (eduflow-deploy 폴더)
copy .env.example .env
notepad .env

메모장이 열리면 아래 한 줄을 찾아서 본인의 API 키로 수정하고 저장하세요(Ctrl+S).

.env
LOCAL_MODE=true
ANTHROPIC_API_KEY=sk-ant-여기에-본인-키-입력
# OPENAI_API_KEY=sk-...
# GOOGLE_API_KEY=...
# UPSTAGE_API_KEY=up_...
API 키 발급처
  • console.anthropic.com — Claude
  • platform.openai.com — GPT
  • aistudio.google.com/apikey — Gemini
  • console.upstage.ai — Solar
최소 1개만 있어도 시작할 수 있어요.
LOCAL_MODE란?
LOCAL_MODE=true가 있으면 Google 로그인·관리자 승인 없이 바로 사용할 수 있어요. 프로젝트 제한도, 모델 제한도 없습니다. 혼자 쓰시는 분을 위한 모드입니다.
5

실행

명령 프롬프트 (eduflow-deploy 폴더)
npm run dev

아래와 같은 메시지가 보이면 성공입니다.

정상 실행 메시지
[EduFlow] LOCAL_MODE 활성화 — 인증 없이 실행
[EduFlow] 서버 실행 중: http://localhost:7829
  ➜  Local:   http://localhost:7830/

크롬·엣지의 주소창(검색창 아님)에 http://localhost:7830을 입력하면 홈 화면이 뜹니다.

Ongoing

다음 실행부터

두 줄이면 충분해요.

명령 프롬프트
cd %USERPROFILE%\Desktop\eduflow-deploy
npm run dev
바탕화면 바로가기 만들기 (선택)

한 번만 만들어두면 더블클릭으로 에듀플로를 시작할 수 있어요.

  1. 바탕화면 우클릭 → 새로 만들기 → 바로 가기
  2. "항목 위치 입력"에 아래를 붙여넣기 
    cmd /k "cd %USERPROFILE%\Desktop\eduflow-deploy && npm run dev"
  3. 이름을 에듀플로 실행으로 저장

종료: 명령 프롬프트에서 Ctrl + C → Y → Enter

업데이트
cd %USERPROFILE%\Desktop\eduflow-deploy
git pull
npm install
npm run dev

projects/ 폴더(내 교재 데이터)는 업데이트로 삭제되지 않아요.

Part 2 · How to use

에듀플로 사용법

여섯 단계로 아이디어에서 완성 교재까지. 교사가 결정하고, AI는 도구입니다.

오픈소스 교육자료 플랫폼이란?

에듀플로는 단순한 AI 교재 생성기가 아닙니다. 교사가 창작자이고, AI는 도구라는 원칙 위에 세워진 플랫폼입니다.

  • 교사가 결정합니다 — AI는 제안하고, 모든 단계에서 수정·거부·재생성할 권한은 선생님에게 있습니다.
  • 끊기지 않는 흐름 — 아이디어 구상부터 웹 교재 배포까지, 한 도구 안에서 완결됩니다.
  • 특정 AI에 종속되지 않습니다 — Claude · GPT · Gemini · Solar 자유 선택, 더 나은 AI가 나오면 전환.
  • 공유하면 더 좋아집니다 — 배포된 교재는 동료 검토를 받고, 학생에게는 학습 플랫폼이, 선생님에게는 포트폴리오가 됩니다.
  • 과정이 투명합니다 — 발행인·검토자·발행일이 교재에 명시되어 신뢰도를 높입니다.

6단계 워크플로우

아이디어에서 배포까지.

01
방향성 논의
AI와 대화하며 교재의 목적·대상·범위를 구체화합니다.
02
목차 작성
AI가 자동 생성한 목차를 검토하고 수정합니다.
03
피드백 & 확정
목차를 최종 확정하고 AI에게 방향을 전달합니다.
04
챕터 생성
전체 배치 또는 챕터별로 본문을 자동 생성합니다.
05
배포
웹사이트(Astro Starlight) 또는 Word 문서로 내보냅니다.
06
포트폴리오
완성된 교재를 모아 포트폴리오로 관리합니다.

프로젝트 만들기

  1. 프로젝트 관리 메뉴 클릭
  2. 프로젝트 ID: 영문+숫자, 날짜-주제 형식 추천 (예: 240315-ai-class)
  3. 제목: 교재 제목 입력
  4. 교과 전문성(WHAT): 수학·과학 / 프로그래밍 / 외국어 / 토론·탐구
  5. 교육 모델(HOW): 교과서 / 인터랙티브 수업 교재 / 자기주도 학습서
  6. 기능 옵션: Mermaid · 수식(KaTeX) · 코드 블록 등
  7. 프로젝트 만들기 클릭
2축 템플릿 시스템

WHAT × HOW 조합으로 교과에 맞는 교재를 생성합니다.

수학·과학 이론 × 교과서 → 탐구 중심, 수식 포함 교과서
프로그래밍 × 자기주도 학습서 → 체크포인트·실습 위주 독학 교재

참고자료 활용

교과서 원문, 교육과정 문서, 수업 자료를 업로드하면 AI가 교재 생성 시 참고합니다.

  • 지원 형식 — PDF · DOCX · XLSX · HWP · HWPX · HTML · TXT · MD · CSV
  • 파일 업로드 — 프로젝트 관리 → 참고자료 탭 → 파일 선택
  • 텍스트 붙여넣기 — 웹에서 복사한 내용을 직접 붙여넣어 저장 가능

교재 배포하기

방법 1 · 웹사이트(Astro Starlight) — 배포 관리 → 웹사이트 빌드. GitHub 계정 연결 시 greatsong.github.io/프로젝트명으로 자동 공개 배포. Mermaid · KaTeX · 인터랙티브 HTML 지원.

방법 2 · Word 문서(DOCX) — 배포 관리 → DOCX 생성 → 다운로드.

내 교재 데이터 위치

파일 탐색기
바탕화면\eduflow-deploy\projects\
├── 프로젝트ID\
│   ├── config.json       # 설정
│   ├── toc.json          # 목차
│   ├── docs\             # 챕터 파일
│   └── discussions\      # AI 대화 기록
백업 권장
projects/ 폴더를 USB나 클라우드에 정기적으로 복사해두세요.

검토와 발행

에듀플로로 만든 교재는 발행인·검토자·발행일이 기록됩니다. 이는 교재의 신뢰도를 높이고, 교사 커뮤니티 안에서 교재가 검증되는 과정을 투명하게 보여줍니다.

검토가 왜 중요한가요?
AI가 생성한 교재는 초안입니다. 동료 교사의 검토를 거치면 교과 정확성·학생 수준 적합성·교육 효과를 확인할 수 있습니다. 검토자 이름이 교재에 명시되면, 그 교재는 개인 작업물이 아니라 교사 커뮤니티가 함께 만든 교육자료가 됩니다.

검토인 등록: 프로젝트 폴더의 config.json에 publishing 항목을 추가합니다.

projects/프로젝트ID/config.json
{
  "title": "내 교재 제목",
  "publishing": {
    "publisher": "홍길동",
    "reviewers": ["김검토", "이리뷰"],
    "published_date": "2026-03-28"
  }
}
  • publisher — 발행인 (교재를 만든 선생님)
  • reviewers — 검토자 목록 (동료 교사, 교과 전문가)
  • published_date — 발행일

이 정보는 배포된 웹사이트 교재 하단과 GitHub 저장소 README에 자동으로 표시됩니다.

Part 3 · FAQ

자주 묻는 질문

각 질문을 클릭하면 답변이 펼쳐집니다.

"API 키가 필요합니다" 버튼이 계속 보여요 ▾

좌측 사이드바 하단의 🔑 AI API 키 버튼 클릭 → 프로바이더 선택 → 키 붙여넣기(Ctrl+V) → 저장.

버튼 우측에 설정됨 뱃지가 초록색으로 바뀌면 정상입니다.

"port already in use" 오류가 납니다 ▾

이전 실행이 종료되지 않은 상태예요. 새 명령 프롬프트에서:

taskkill /F /IM node.exe

이후 npm run dev을 다시 실행하세요.

npm install 중 오류가 납니다 ▾ 
node --version    # v20 이상인지 확인
npm cache clean --force
npm install

그래도 안 되면 eduflow-deploy 폴더를 삭제하고 Step 3부터 다시 실행하세요.

localhost:7830에 아무것도 안 보입니다 ▾
  • 명령 프롬프트에서 Local: http://localhost:7830/ 메시지가 보일 때까지 30초 기다리세요.
  • 주소창에 직접 입력하세요 (검색창이 아닌 브라우저 맨 위 주소창).
  • 방화벽 팝업이 뜨면 허용을 클릭하세요.
"지정된 경로를 찾을 수 없습니다" 오류 ▾

OneDrive가 바탕화면을 동기화 중일 때 생깁니다.

cd "%USERPROFILE%\OneDrive\Desktop"
# 또는 한국어 경로
cd "%USERPROFILE%\OneDrive\바탕 화면"
챕터 내용이 너무 짧거나 부실합니다 ▾
  • 모델을 Claude Sonnet 4.6 이상으로 변경
  • 방향성 논의(Step 1)에서 대상 독자·깊이·특별 요구사항을 더 구체적으로 전달
  • 프로젝트 관리 → 프롬프트 설정에서 추가 지침 입력
  • 챕터별로 재생성 버튼 클릭
에듀플로를 업데이트하고 싶습니다 ▾ 
cd %USERPROFILE%\Desktop\eduflow-deploy
git pull
npm install
npm run dev

기존 프로젝트(교재 데이터)는 그대로 유지됩니다.

eduflow-js 리포에서 이사 오려면? ▾

데이터 구조가 동일해서 프로젝트 폴더를 그대로 복사하면 됩니다.

  1. 기존 eduflow-js\projects\ 안의 프로젝트 폴더들을 복사
  2. 새 eduflow-deploy\projects\에 붙여넣기
  3. npm run dev로 실행하면 그대로 열립니다

API 키는 방식이 동일(.env 또는 브라우저 입력)하니 기존 키를 다시 넣어주세요.
