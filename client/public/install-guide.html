1 Git 설치하기

  1. git-scm.com/download/win 접속 → 64-bit Git for Windows Setup 다운로드
  2. 설치 파일 실행 → 모든 화면에서 Next → Install → Finish
  3. 설치 확인: 시작 메뉴 → cmd 검색 → 명령 프롬프트 열기
명령 프롬프트
git --version
# 출력 예시: git version 2.47.1.windows.1

2 Node.js 설치하기

  1. nodejs.org 접속 → 초록색 LTS 버튼 클릭하여 다운로드
  2. 설치 파일 실행 → Next → 동의 체크 → Install → Finish
  3. 새로운 명령 프롬프트를 열고 확인
명령 프롬프트 (새로 열기)
node --version
# 출력 예시: v22.13.0
npm --version
# 출력 예시: 10.9.2

3 에듀플로 다운로드

명령 프롬프트
cd %USERPROFILE%\Desktop
git clone https://github.com/greatsong/eduflow-deploy.git
cd eduflow-deploy
npm install
ℹ️ 안내
  • npm install은 첫 실행 시 1~3분 소요됩니다 (패키지 설치)
  • 바탕화면에 eduflow-deploy 폴더가 생성됩니다
  • OneDrive 바탕화면이면: cd "%USERPROFILE%\OneDrive\Desktop" 또는 cd "%USERPROFILE%\OneDrive\바탕 화면"

4 API 키 설정

에듀플로 폴더 안에 .env.example 파일을 복사해서 .env를 만듭니다.

명령 프롬프트 (eduflow-deploy 폴더)
copy .env.example .env

.env 파일을 메모장으로 열어서 API 키를 입력합니다.

명령 프롬프트
notepad .env
✅ .env 파일에서 이 줄만 수정하세요
# 본인의 API 키로 변경 (최소 하나)
ANTHROPIC_API_KEY=sk-ant-여기에-본인-키-입력

Claude 키: console.anthropic.com · GPT 키: platform.openai.com · Gemini 키: aistudio.google.com

ℹ️ LOCAL_MODE란?

.env 파일에 LOCAL_MODE=true가 기본 설정되어 있어 Google 로그인 없이 바로 사용할 수 있습니다. 모든 AI 모델과 기능이 제한 없이 개방됩니다.

5 실행

명령 프롬프트 (eduflow-deploy 폴더)
npm run dev
✅ 정상 실행 메시지
[0] [EduFlow] LOCAL_MODE 활성화 — 인증 없이 실행
[0] [EduFlow] 서버 실행 중: http://localhost:7829
[1]   ➜  Local:   http://localhost:7830/

브라우저에서 http://localhost:7830 접속하면 바로 사용할 수 있습니다.

🔁 이후 재실행 방법

명령 프롬프트
cd %USERPROFILE%\Desktop\eduflow-deploy
npm run dev

바탕화면 바로가기 만들기: 바탕화면 우클릭 → 새로 만들기 → 바로 가기
위치: cmd /k "cd %USERPROFILE%\Desktop\eduflow-deploy && npm run dev"
이름: 에듀플로 실행

종료: 명령 프롬프트에서 Ctrl + C → Y → Enter

🔄 업데이트 방법
cd %USERPROFILE%\Desktop\eduflow-deploy
git pull
npm install
npm run dev

💡 업데이트해도 projects/ 폴더(내 교재 데이터)는 삭제되지 않습니다.

PART 2 — 에듀플로 사용법
6단계 워크플로우로 아이디어 → 완성 교재까지
💡 "오픈소스 교육자료 플랫폼"이란?

에듀플로는 단순한 AI 교재 생성기가 아닙니다. 교사가 창작자이고, AI는 도구라는 원칙 위에 세워진 플랫폼입니다.

  • 교사가 결정합니다 — AI는 제안하고, 모든 단계에서 수정·거부·재생성할 권한은 선생님에게 있습니다.
  • 끊기지 않는 흐름 — 아이디어 구상부터 학생 손에 닿는 웹 교재 배포까지, 하나의 도구 안에서 완결됩니다.
  • 특정 AI에 종속되지 않습니다 — Claude, GPT, Gemini, Solar 중 자유롭게 선택하고, 더 나은 AI가 나오면 언제든 전환할 수 있습니다.
  • 공유하면 더 좋아집니다 — 완성된 교재는 웹으로 공개 배포되어 동료 교사의 검토를 받고, 학생에게는 학습 플랫폼이, 선생님에게는 포트폴리오가 됩니다.
  • 과정이 투명합니다 — 누가 만들고, 누가 검토했는지가 교재에 명시됩니다. 발행인·검토자·발행일이 기록되어 교재의 신뢰도를 높입니다.

"좋은 수업 아이디어를 체계적인 교육자료로" — 에듀플로가 추구하는 것은 교육 콘텐츠 제작의 민주화입니다.

📁 프로젝트 생성하기

  1. 프로젝트 관리 메뉴 클릭
  2. 프로젝트 ID: 영문+숫자, 날짜-주제 형식 추천 (예: 240315-ai-class)
  3. 제목: 교재 제목 입력
  4. 교과 전문성(WHAT): 수학·과학 / 프로그래밍 / 외국어 / 토론·탐구 등 선택
  5. 교육 모델(HOW): 교과서 / 인터랙티브 수업 교재 / 자기주도 학습서 등 선택
  6. 기능 옵션: Mermaid 다이어그램, 수식(KaTeX), 코드 블록 등 원하는 기능 선택
  7. 🚀 프로젝트 만들기 클릭
💡 2축 템플릿 시스템

WHAT(교과 전문성) × HOW(교육 모델)을 조합하면 어떤 교과든 최적의 교재를 생성할 수 있습니다.
예: 수학·과학 이론 × 교과서 → 탐구 중심, 수식 포함 교과서
예: 프로그래밍 × 자기주도 학습서 → 체크포인트, 실습 위주 독학 교재

📎 참고자료 활용

교과서 원문, 교육과정 문서, 수업 자료를 업로드하면 AI가 교재 생성 시 참고합니다.

  • 지원 형식: PDF · DOCX · XLSX · HWP · HWPX · HTML · TXT · MD · CSV
  • 파일 업로드: 프로젝트 관리 → 참고자료 탭 → 파일 선택
  • 텍스트 복붙: 웹에서 복사한 내용을 직접 붙여넣어 저장 가능

🌐 교재 배포하기

방법 1: MkDocs 웹사이트

배포 관리 → MkDocs 빌드 → 로컬 미리보기 또는 GitHub Pages 배포

  • GitHub 계정 연결 시 자동으로 greatsong.github.io/프로젝트명 형태로 공개 배포
  • Mermaid 다이어그램, KaTeX 수식, 인터랙티브 HTML 모두 지원

방법 2: Word 문서(DOCX)

배포 관리 → DOCX 생성 → 다운로드 → 수업용 자료로 배포

내 교재 데이터 위치

│   ├── docs\               ← 챕터 파일

🔍 검토와 발행

에듀플로로 만든 교재는 발행인·검토자·발행일이 기록됩니다. 이는 교재의 신뢰도를 높이고, 교사 커뮤니티 안에서 교재가 검증되는 과정을 투명하게 보여줍니다.

📋 검토가 왜 중요한가요?

AI가 생성한 교재는 초안입니다. 동료 교사의 검토를 거치면 교과 정확성, 학생 수준 적합성, 교육 효과를 확인할 수 있습니다. 검토자 이름이 교재에 명시되면, 그 교재는 개인 작업물이 아니라 교사 커뮤니티가 함께 만든 교육자료가 됩니다.

검토인 등록 방법

프로젝트 폴더의 config.json 파일에서 publishing 항목을 추가합니다.

projects/프로젝트ID/config.json
{
  "title": "내 교재 제목",
  "publishing": {
    "publisher": "홍길동",
    "reviewers": ["김검토", "이리뷰"],
    "published_date": "2026-03-28"
  }
}
  • publisher: 발행인 (교재를 만든 선생님)
  • reviewers: 검토자 목록 (동료 교사, 교과 전문가 등)
  • published_date: 발행일

이 정보는 MkDocs 교재 하단과 GitHub 저장소의 README에 자동으로 표시됩니다.

✅ 검토 워크플로우 예시
  1. 선생님이 에듀플로로 교재를 만들고 GitHub Pages로 배포합니다.
  2. 배포된 URL을 동료 교사에게 공유하여 검토를 요청합니다.
  3. 검토 의견을 반영하여 챕터를 수정하고 재배포합니다.
  4. config.json에 검토자 이름을 추가합니다.
  5. 다시 배포하면 교재에 검토자 정보가 표시됩니다.

PART 3 — 자주 묻는 질문
클릭하여 답변 확인
🔑 "API 키가 필요합니다" 버튼이 계속 보여요 ▾

좌측 하단의 🔑 AI API 키 버튼 클릭 → API 키 붙여넣기(Ctrl+V) → 저장 버튼 클릭

버튼이 ✅ 설정됨으로 바뀌면 정상입니다.

🔴 "port already in use" 오류가 납니다 ▾

이전 실행이 종료되지 않았습니다. 새 명령 프롬프트에서:

taskkill /F /IM node.exe

이후 다시 npm run dev 실행

🔴 npm install 중 오류가 납니다 ▾ 
node --version    # v20 이상인지 확인
npm cache clean --force
npm install

그래도 안 되면 에듀플로 폴더를 삭제하고 Step 3부터 다시 실행하세요.

🌐 브라우저에서 localhost:7830에 아무것도 안 보입니다 ▾
  • 명령 프롬프트에서 Local: http://localhost:7830/ 메시지가 보일 때까지 30초 기다리기
  • 주소창에 직접 입력 (검색창이 아닌 주소창에)
  • 방화벽 팝업이 나오면 허용 클릭
📁 "지정된 경로를 찾을 수 없습니다" 오류 ▾

OneDrive 연동 바탕화면이라면:

cd "%USERPROFILE%\OneDrive\Desktop"
# 또는
cd "%USERPROFILE%\OneDrive\바탕 화면"
🤖 챕터 내용이 너무 짧거나 부실합니다 ▾
  • 모델을 Claude Sonnet 4.6 이상으로 변경
  • 방향성 논의(Step 1)에서 대상 독자, 깊이, 특별 요구사항을 더 구체적으로 이야기하기
  • 프로젝트 관리 → 프롬프트 설정에서 추가 지침 입력
  • 챕터별로 재생성 버튼 클릭
🔄 에듀플로를 업데이트하고 싶습니다 ▾ 
cd %USERPROFILE%\Desktop\eduflow-deploy
git pull
npm install
npm run dev

💡 기존 프로젝트(교재 데이터)는 그대로 유지됩니다.
