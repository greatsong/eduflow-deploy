## 들어가며 — "물리·게임·시뮬은 다 이 두 줄로 굴러간다"

농구공 포물선, 행성 궤도, 게임 총알. 코드는 놀랄 만큼 비슷합니다.

```
속도 = 속도 + 가속도 * dt
위치 = 위치 + 속도 * dt
```

이 두 줄이 **모든 운동 시뮬의 심장**입니다. 던지기·행성·입자 효과의 출발점.

## 핵심 개념 — 벡터 덧셈과 `dt`

`속도 = vector(2, 3, 0)`은 *"오른쪽 초당 2, 위로 초당 3"* 한 묶음. 여기에 `중력 = vector(0, -9.8, 0)`을 매 순간 살짝 더하면 솟던 속도가 줄어 결국 떨어집니다. 이 *"조금씩 더한다"*가 가속도의 정의예요.

`dt = 0.02`는 *"한 프레임은 0.02초"*. 작을수록 정밀(0.001=행성·진자), 클수록 거침(0.1=슬로우). 표준은 0.02.

## 🧰 베이스라인으로 가져갈 때 바꿀 곳

| ★ 위치 | 무엇을 바꾸나 | 결과 변화 |
|---|---|---|
| `pos=vector(-3,3,0)` | 출발 위치 | 발사대·시작점 |
| `속도 = vector(2,3,0)` | 초기 속도 | 던지는 방향·세기 |
| `중력 = vector(0,-9.8,0)` | 가속도 | 0이면 무중력, 위로면 풍선 |
| `dt = 0.02` | 시간 한 조각 | 정밀도와 속도의 균형 |
| `while t < 1.5:` | 시뮬 시간 | 짧으면 끊기고, 길면 화면 밖 |
| `make_trail=True` | 자취 | 궤적 시각화 |

## ⚠️ 자주 하는 실수

!!! warning "공이 화면 밖으로 사라질 때"
    1. 속도가 너무 커 즉시 사라짐 — `vector(2, 3, 0)`부터.
    2. 중력 부호 실수 — `vector(0, 9.8, 0)`이면 위로 떨어집니다.
    3. `dt`가 너무 크면(0.5+) 한 프레임에 너무 많이 움직여 *"순간이동"*처럼.
    4. `중력 * dt` 곱셈 누락 → 1프레임에 가속도 통째로 더해져 폭주.
