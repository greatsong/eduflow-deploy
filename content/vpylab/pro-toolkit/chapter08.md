## 🎯 이 차시의 학습 목표

- [ ] `graph(...)` + `gcurve(...)` 두 줄로 2D 그래프 창을 만들 수 있다
- [ ] `.plot(x, y)`가 점을 누적해 곡선을 그린다는 흐름을 안다
- [ ] 이 베이스라인을 내 프로젝트에 가져갈 때 ★ 마커 위치를 모두 안다

## 들어가며 — "결과를 그래프로 보여주면 설득이 끝난다"

4차시에서 던진 공의 높이를 숫자로 듣고 싶은 사람은 없습니다. **그래프로 보여줘야 한 번에 이해됩니다.** 이 차시는 *"숫자가 흐르면 그래프가 그려진다"*를 가장 짧게 보여주는 시드 — 시뮬 결과 가시화, 센서 데이터, 함수 비교의 출발점.

## 핵심 개념 — 별도 창 + 점 누적

`g = graph(title='...', xtitle='x', ytitle='y', width=600, height=380)`은 3D 무대와 **다른 창**을 띄웁니다. *"왼쪽엔 공이 날아가고, 오른쪽엔 그 높이가 그려진다"*가 가능해요.

`선 = gcurve(color=color.cyan)` 후 `선.plot(t, math.sin(t))`. `gcurve`는 6차시 `curve`의 2D 형제 — `.plot(x, y)`마다 점이 추가되고 자동으로 이어집니다.

| 도구 | 어디에 | 좌표 |
|---|---|---|
| `curve` | 3D 무대 | x, y, z |
| `gcurve` | 그래프 창 | x, y |

## 🧰 베이스라인으로 가져갈 때 바꿀 곳

| ★ 위치 | 무엇을 바꾸나 | 결과 변화 |
|---|---|---|
| `title='...'` | 그래프 제목 | 발표 자료에 그대로 |
| `xtitle/ytitle` | 축 이름 | 시간(s)·높이(m) |
| `width/height` | 창 크기 | 발표 화면 맞춤 |
| `gcurve`의 `color` | 곡선 색 | 여러 시리즈 비교 |
| `while t < 6.28:` | x축 범위 | 1주기↔여러 주기 |
| `선.plot(t, ...)` | 그릴 함수 | 어떤 양 |

## ⚠️ 자주 하는 실수

!!! warning "그래프가 안 그려지거나 이상할 때"
    1. 변수명을 `gcurve = gcurve(...)`로 두면 다음 호출이 깨짐 — `선 = gcurve(...)`.
    2. `.plot(x, y)`에 인자 1개만 — 반드시 둘.
    3. 루프 안에서 `gcurve`를 매번 새로 만들면 점이 1개만 — 루프 **밖**에서 한 번.
    4. `t` 범위가 너무 좁아 점이 2~3개뿐.
