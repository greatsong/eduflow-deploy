## 🎯 이 차시의 학습 목표

- [ ] `slider(...)`로 위젯을 만들고 콜백 함수로 객체에 연결할 수 있다
- [ ] `scene.bind('keydown', 함수)`로 키보드 입력을 받을 수 있다
- [ ] 이 베이스라인을 내 프로젝트에 가져갈 때 ★ 마커 위치를 모두 안다

## 들어가며 — "사용자가 만지면 작품이 완성된다"

지금까지 10개 베이스라인으로 그림·운동·소리까지 다 넣었습니다. 마지막은 ***보는 사람*이 *만지는 사람*이 되게** 만드는 일. 슬라이더와 키보드 둘이면 게임도 체험형 시연도 됩니다.

## 핵심 개념 — 슬라이더 콜백 + 키보드 이벤트

`slider(min=0.1, max=2.0, value=0.4, bind=크기바꾸기)` — `bind=크기바꾸기`는 *"슬라이더가 움직일 때마다 이 함수를 불러"*. 함수 안 `s.value`가 슬라이더 현재 값. 끌면 함수가 자동 호출되고 공이 즉시 변합니다.

`scene.bind('keydown', 키눌림)` — `evt.key`가 어떤 키인지 알려줍니다(`'left'`, `'right'`, `'a'`, `'space'` 등).

| 입력 | 함수에서 받는 값 |
|---|---|
| 슬라이더 | `s.value` (숫자) |
| 키보드 | `evt.key` (문자열) |

| ★ 위치 | 무엇을 바꾸나 | 결과 변화 |
|---|---|---|
| `공.radius = s.value` | 어떤 속성을 바꿀지 | 색·위치·속도·중력 |
| `slider(min, max)` | 값 범위 | 허용 범위 |
| `slider`의 `value` | 시작값 | 처음 화면 |
| `이동 = 0.2` | 키 한 번당 이동량 | 빠른↔정밀 |
| `if evt.key == 'left':` | 어떤 키 → 어떤 일 | 조작 키 설계 |

## 🧩 마지막 단계 — 베이스라인 2~3개 합치기

베이스라인 11개를 *합쳐서* 자기 작품을 만들 차례입니다.

**시나리오: "행성 탐험" 미니 데모 (60줄 안)** — 1차시 골격(검은 우주·시간 루프) + 3차시 조명(태양빛 + 어두운 배경) + 5차시 회전(행성 공전 + 궤적) + 11차시 인터랙션(슬라이더로 공전 속도, 화살표로 시야) + 10차시 사운드(시작 *도-미-솔* 화음). 다섯 시드를 골라 ★만 바꾸면 60줄 안에 완성됩니다.

*"이거 그냥 시드 다섯 개 합친 건데?"*가 이 코스의 목적입니다. **베이스라인은 부담 없는 출발점**, 자기 색은 ★를 어떻게 비트느냐로 드러납니다.

## ⚠️ 자주 하는 실수

!!! warning "인터랙션이 안 먹힐 때"
    1. `bind=크기바꾸기()`처럼 괄호 붙임 — 함수 *자체*를 넘겨야 합니다.
    2. 화면을 클릭하지 않아 키 입력 무시 — 마우스 클릭이 *포커스*를 줍니다.
    3. 콜백 안 `print` 남발로 화면이 멈춰 보임.
    4. 슬라이더 `min > max`.
