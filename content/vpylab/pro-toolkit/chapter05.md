## 🎯 이 차시의 학습 목표

- [ ] `.rotate(angle, axis, origin)` 세 인자의 역할을 안다
- [ ] `origin`을 바꾸면 자전과 공전이 갈린다는 것을 안다
- [ ] 이 베이스라인을 내 프로젝트에 가져갈 때 ★ 마커 위치를 모두 안다

## 들어가며 — "도는 것은 거의 다 한 줄로 된다"

행성 공전, 풍차, 시계 바늘, 자동차 바퀴. 다 달라 보이지만 시작은 같은 한 줄입니다.

```
객체.rotate(angle=각도, axis=축, origin=중심점)
```

이 시드 하나로 공전·자전·풍차·기어 모든 회전 작품의 출발점이 됩니다.

## 핵심 개념 — 회전 3요소와 `origin` 트릭

`angle`은 1프레임당 도는 각(라디안), `axis`는 회전축, `origin`은 회전 중심. `angle=0.03`은 약 1.7도 — `rate(60)`과 합치면 1초에 100도쯤. 빠르면 0.01, 느리면 0.05.

같은 `.rotate(...)`인데 `origin`만 바꾸면 의미가 정반대가 됩니다.

- `origin=행성.pos` → **자전** (자기 자리에서 빙글)
- `origin=vector(0, 0, 0)` → **공전** (다른 점을 중심으로)

코드 한 줄 차이로 "팽이"가 "달"이 됩니다.

## 🧰 베이스라인으로 가져갈 때 바꿀 곳

| ★ 위치 | 무엇을 바꾸나 | 결과 변화 |
|---|---|---|
| `pos=vector(3,0,0)` | 출발 자리 | 궤도 반지름 |
| `angle=0.03` | 회전 속도 | 1년이 짧거나 길어짐 |
| `axis=vector(0,1,0)` | 회전축 | 수평·수직·기울어진 궤도 |
| `origin=vector(0,0,0)` | 회전 중심 | 항성 위치 또는 자전축 |
| `make_trail=True` | 자취 | 공전 궤적 |

## ⚠️ 자주 하는 실수

!!! warning "회전이 이상해 보일 때"
    1. `angle`이 너무 커서(0.5) 한 프레임에 30도 점프.
    2. `axis=vector(0,0,0)` — 축이 없어 회전이 안 일어남.
    3. `origin` 누락 → 기본은 자전. *공전 의도였는데 안 움직여 보임*.
    4. 자취 `radius`가 너무 작아 안 보임 — 0.04~0.1.
