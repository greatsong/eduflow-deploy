## 🎯 이 차시의 학습 목표

- [ ] `for i in range(5):`의 의미를 *"같은 일을 5번 시킨다"*로 풀어 말할 수 있다
- [ ] 반복문 안의 `i`가 0, 1, 2, 3, 4로 바뀌어 들어간다는 사실을 안다
- [ ] 한 줄짜리 `sphere(...)`를 다섯 개로 늘리는 *두 가지 방법*(복붙 vs 반복문)의 차이를 설명할 수 있다

## 들어가며 — "같은 줄을 다섯 번 적을래?"

선생님이 칠판에 *"안녕하세요"*를 다섯 번 적으라고 했다고 상상해 봐요. 보통은 한 줄, 또 한 줄, 또 한 줄 — 다섯 번을 손으로 적습니다. 그런데 친구가 이런 말을 합니다. *"그냥 '5번 적어'라고만 말하면 안 돼?"*

코딩에서는 그게 됩니다. 한 줄짜리 부탁에 *"이걸 5번 해 줘"*라는 포장만 씌우면, 컴퓨터가 알아서 다섯 번을 반복해요. 오늘은 입문1에서 한 개씩 띄웠던 공을 **한 줄로 다섯 개** 띄워 봅니다.

## 핵심 개념 1 — `for i in range(5):` 라는 포장

이 차시 코드의 핵심은 다음 두 줄입니다.

> "i를 0부터 시작해서 5번 바꿔 가며, 아래 들여쓴 줄을 그때마다 실행해 줘."

`for`는 *"반복할게"* 라는 약속이고, `range(5)`는 *"5번 = 0,1,2,3,4"* 라는 횟수표예요. 그리고 `i`는 그 **이번 차례 번호**가 잠깐 머무르는 자리입니다.

| 차례 | `i`의 값 | 들여쓴 줄에서 실제로 실행되는 것 |
|---|---|---|
| 1번째 | 0 | `sphere(pos=vector(0 * 1.2, 0, 0), ...)` → 가운데 |
| 2번째 | 1 | `sphere(pos=vector(1 * 1.2, 0, 0), ...)` → 오른쪽으로 1.2만큼 |
| 3번째 | 2 | `sphere(pos=vector(2 * 1.2, 0, 0), ...)` |
| 4번째 | 3 | `sphere(pos=vector(3 * 1.2, 0, 0), ...)` |
| 5번째 | 4 | `sphere(pos=vector(4 * 1.2, 0, 0), ...)` |

같은 코드 한 줄이 **다섯 번 다른 위치에서** 실행됩니다. 위치가 달라지는 이유는 `i * 1.2` 부분에서 `i`가 매번 0, 1, 2, 3, 4로 바뀌기 때문이에요.

## 핵심 개념 2 — 들여쓰기가 곧 "반복할 범위"

파이썬에는 *"여기부터 여기까지 반복해"* 라는 괄호가 따로 없어요. 대신 **들여쓰기(스페이스 4칸)**로 표시합니다.

> `for ...:` 다음 줄부터 **들여쓰기된 줄까지**가 반복 대상.

들여쓰기가 풀린 줄은 반복문 *바깥*이 됩니다. 들여쓰기 한 칸을 잘못 빼면, 5번 만들고 싶었던 공이 1개만 만들어지고 끝나는 사고가 자주 일어나요.

## ⚠️ 자주 하는 실수

!!! warning "초보자 3대 함정"
    1. **콜론(`:`)을 빼먹는다** — `for i in range(5)` 뒤에는 반드시 `:`이 있어야 합니다.
    2. **들여쓰기를 안 한다** — `for i in range(5):` 다음 줄이 들여쓰기 없으면 *"반복할 게 없네?"* 라며 에러가 나요.
    3. **`range(5)`는 5까지가 아니라 4까지** — 0, 1, 2, 3, 4 다섯 개입니다. *시작은 0, 끝은 5 미만*이라고 외워 두세요.

## 🧑‍🏫 교사 노트

!!! note "수업 운영 팁 — 반복문 첫 만남 5분 도입"
    "여러분, 만약 공 100개를 화면에 띄워야 한다면 `sphere(...)` 줄을 100번 복붙할 건가요? 50번째쯤에서 한 글자 틀리면 어떻게 찾을까요? 오늘은 **한 줄로 100개**를 만드는 마법을 배웁니다."

    - 학생들이 `range(5)`의 결과를 *"1부터 5까지"* 라고 오해하기 쉽습니다. **0부터 시작, 5는 포함 안 됨**을 칠판에 크게 적어 두세요.
    - `i` 값이 어떻게 바뀌는지 보고 싶다면, `print(i)` 한 줄을 들여쓰기 안에 추가해 콘솔로 확인하게 하면 *"아!"* 순간이 옵니다.
    - 짝활동: 옆 친구에게 *"i가 5번째일 때 공은 어디에 있어?"*를 물어보고 답해보게 하세요.
