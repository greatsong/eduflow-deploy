# 에듀플로 사용 가이드 (선생님용)

**처음 여시는 분도 어렵지 않게, 익숙한 분은 빠르게 넘기실 수 있도록** 정리했어요. 교재 한 권을 처음부터 끝까지 만드는 흐름을 따라 설명드립니다.

> 에듀플로가 어떻게 돌아가는지 간단히 말하면 — 선생님이 "이런 수업을 하고 싶어요"라고 던지시면, AI가 방향 잡기 → 목차 → 본문 → 이미지·다이어그램 → 웹 교재까지 쭉 이어서 만들어드려요. 중간중간 "이건 다시" 하시면 그대로 따라갑니다.

---

## 처음 접속하실 때

도메인 주소를 받으셨으면 접속하세요. (혼자 노트북에서 쓰실 때는 http://localhost:7830 )

### 로그인 (공용 서버인 경우)
1. **Google 계정으로 로그인** 버튼 클릭
2. 처음이시면 간단한 지원 정보(이름, 소속, 과목 등) 입력
3. 관리자 승인이 필요한 서비스라면 잠깐 대기하셨다가 알림 받으시면 돼요

### API 키 (로컬·개인 모드인 경우)

좌측 사이드바에 **🔑 AI API 키** 버튼이 있어요. 한 번만 입력해두시면 브라우저가 기억합니다. (키 발급처는 [LOCAL.md](./LOCAL.md)에 정리돼 있어요)

> 관리자가 운영하는 서버에서는 이 단계가 보통 생략돼요 — 관리자가 키를 넣어뒀을 거예요.

## 6단계 워크플로우 한눈에

```
Step 0. 프로젝트 관리     — 새 교재 만들기, 템플릿 고르기, 참고자료 올리기
  ↓
Step 1. 방향성 논의       — AI와 대화하며 수업 방향 잡기
  ↓
Step 2. 목차 작성         — AI가 목차 뽑고, 고치고 싶은 곳 직접 수정
  ↓
Step 3. 피드백·확정       — 한 번 더 다듬고 "이대로 갑시다" 버튼
  ↓
Step 4. 챕터 제작         — 본문 + 이미지·다이어그램 생성 (한꺼번에 or 하나씩)
  ↓
Step 5. 배포              — 웹사이트·Word·GitHub Pages로 공유
```

각 단계는 **왼쪽에서 오른쪽으로 한 번만 가는 길이 아니에요.** 언제든 뒤로 돌아가서 고치실 수 있어요.

---

## Step 0. 프로젝트 관리

새 교재 하나를 시작하는 단계예요. 세 가지 경로 중 편한 걸 고르세요.

| 경로 | 언제 쓸지 |
|---|---|
| **빈 프로젝트로 시작** | 내 머릿속에 이미 구성이 있을 때 |
| **템플릿 사용** | "이런 스타일의 교재"라는 큰 틀부터 고르고 싶을 때 |
| **빠른 시작** | 주제만 던지면 AI가 프로젝트 설정부터 목차까지 한 번에 |

### 교육 템플릿 (8종)

템플릿은 단순한 디자인 스타일이 아니라 **교육 모델** 그 자체예요. 각각 다른 철학과 구조를 담고 있어요.

| 템플릿 | 이럴 때 |
|---|---|
| **학교 교과서** | 정규 수업용, 단원 구성이 명확할 때 |
| **프로그래밍 과정** | 코딩 실습 중심, 단계별 실습이 쌓이는 형태 |
| **자기주도 학습서** | 독학 가능하도록, 자기점검이 많이 들어감 |
| **워크샵 교안** | 짧은 시간, 실습·활동 중심 |
| **비즈니스 교육** | 기업 교육, 사례 중심 |
| **교사용 지도서 (4C)** | 4C(Communication·Collaboration·Critical Thinking·Creativity) 기반 교사 가이드 |
| **스토리텔링** | 이야기 흐름으로 개념 전개 |
| **클래스 프리뷰** | 수업 전 학생에게 미리 보여주는 요약형 자료 |

### 2축 템플릿 시스템 (고급)

템플릿을 더 세밀하게 조합하시려면 **교과 영역(WHAT) × 교육 모델(HOW)**로 고르실 수 있어요. 예: "과학(WHAT) × 자기주도 학습(HOW)" 조합.

기능 옵션도 따로 켜고 끌 수 있어요 — 코드 블록, 수식(LaTeX), Mermaid 다이어그램, 회로도, 격식체, 배너·카드 레이아웃 등.

### 참고자료 업로드

기존에 쓰던 수업 자료가 있으시면 올려주세요. AI가 참고해서 교재를 만들어요.

- 지원 형식: PDF, Word(DOCX), 텍스트, 마크다운, 엑셀, 한글(HWP) 등
- 여러 개 올리실 수 있어요

---

## Step 1. 방향성 논의

여기가 에듀플로의 **첫 번째 중요한 순간**이에요. AI랑 대화하시면서 "이 교재가 어떤 느낌이면 좋을까"를 잡아갑니다.

### 대화 예시
```
나: 중학교 2학년 과학 '힘과 운동' 단원을 만들 건데, 
    아이들이 실생활 예시로 먼저 감을 잡고 개념으로 넘어갔으면 해요.

AI: 실생활 예시를 앞에 두는 귀납적 접근이시네요. 
    구체적으로 어떤 예시들을 생각하셨나요? 교통·운동·놀이기구 중에서...
```

- 대화 내용은 **자동 저장**돼요
- 대화가 어느 정도 무르익으면 AI가 요약을 만들어줘요 → 이게 다음 단계의 밑바탕이 됩니다
- 언제든 "이 방향은 다시" 하셔도 되고, 새 대화를 시작하셔도 돼요

---

## Step 2. 목차 작성

논의한 내용을 바탕으로 AI가 **Part → Chapter** 구조의 목차를 자동 생성해요.

### 자동 생성되는 정보
- 각 챕터의 **학습 목표**, **개요**, **예상 소요시간**
- 전체 차시(회차) 구성

### 직접 수정
- 챕터 추가·삭제·순서 변경
- 제목 수정
- "이 챕터는 더 자세히" 같은 메모를 붙여서 다음 단계에 힌트 주기

> 분량이 너무 많이 나올 수 있어요. 그럴 땐 "차시 수를 N차시로 맞춰주세요" 같은 메시지를 AI에 전달하시면 재조정해줘요.

---

## Step 3. 피드백·확정

AI와 한 번 더 목차를 다듬는 단계예요. 마음에 드시면 **"목차 확정"** 버튼. 이 시점부터 마스터 컨텍스트가 잠기고, 이후 챕터 생성이 이 목차를 기준으로 일관되게 이뤄져요.

> 확정 후에도 목차 수정은 가능하지만, 이미 만들어진 챕터와의 일관성은 직접 봐주셔야 해요.

---

## Step 4. 챕터 제작

두 가지 방식이 있어요. 상황 따라 고르세요.

### 방식 A. 배치 생성 (한꺼번에)

**"AI야, 전체 챕터 다 써줘"** — 모든 챕터가 병렬로 생성됩니다.

- 실시간 진행 상황과 예상 비용 표시
- 완료 후 챕터별로 하나씩 열어서 검토·수정
- 10챕터짜리 교재 기준 **5~15분** 정도 걸려요 (모델·동시 실행 수에 따라)

### 방식 B. 인터랙티브 (하나씩 대화하며)

**챕터를 하나씩 열고 AI와 대화하며 만들기.** 각 챕터가 마음에 들 때까지 다듬은 후 다음으로 넘어가요.

- 챕터별 채팅 기록이 저장돼요
- "이 섹션은 예시를 더 넣어주세요", "여기는 너무 길어요" 같은 구체 요청 가능

### 다이어그램

Mermaid 다이어그램이나 회로도는 본문에 섞여서 자동 생성돼요. 클릭하면 라이트박스로 확대 보기 가능.

### 평가 단계 (0~4)

| 단계 | 내용 |
|---|---|
| 0 | 평가 없음 |
| 1 | 자기점검 체크리스트 |
| 2 | 확인 문제 (객관식 + 서술형, 정답 숨김) **← 기본값** |
| 3 | 형성 평가 (난이도별 + 자기점검) |
| 4 | 인터랙티브 퀴즈 (채점·피드백·재도전) |

프로젝트 설정에서 바꾸시면 이후 생성되는 챕터에 반영돼요.

### AI 모델 선택

챕터마다 다른 모델을 써보셔도 돼요.

| 등급 | 대표 모델 | 언제 |
|---|---|---|
| 경제적 | Haiku 4.5, GPT-4.1 mini, Gemini Flash | 빠른 테스트, 초안 |
| 균형 | **Sonnet 4.6, GPT-4.1, Gemini 3.1** | 일반 추천 (기본값) |
| 고품질 | Opus 4.6/4.7, GPT-5.4 | 중요한 교재, 마무리 품질 |

> **팁**: 먼저 경제 모델로 전체 흐름을 뽑아본 뒤, 중요한 챕터만 고품질 모델로 다시 만드는 방식이 가성비 좋아요.

---

## Step 5. 배포

완성된 교재를 학생 손에 닿게 하는 단계예요. 세 가지 방식 모두 한 프로젝트에서 동시에 가능합니다.

### 웹사이트로 배포 (추천)
- **Astro Starlight** 기반 정적 사이트로 빌드 (기본 테마)
- MkDocs Material도 선택 가능 (레거시)
- 사이드바, 검색, 반응형 레이아웃, 다크 모드 전부 포함
- 로컬에서 미리보기 가능

### GitHub Pages로 자동 배포
- GitHub 계정 연결 필요 (Step 5 안내 따라가시면 돼요)
- 배포할 때마다 **포트폴리오**에도 자동 등록 (제목·설명·URL·차시 수)

### Word 문서로 다운로드
- 한 권짜리 DOCX 파일로 변환
- 인쇄용 자료로 활용

## 포트폴리오

완성하신 교재가 자동으로 모이는 공간이에요. Home 또는 포트폴리오 탭에서 확인.

- 교재별 카드(제목·설명·차시 수·배포 URL)
- 전체 통계
- 각 교재의 상세 리포트 (대화 기록, AI 사용량, 배포 이력)

공개 포트폴리오 예시: [에듀플로 포트폴리오](https://greatsong.github.io/eduflow-portfolio/)

## AI 모델 비교 탭

같은 프롬프트로 여러 AI 모델의 답을 **동시에 받아서 비교**할 수 있는 실험실 같은 공간이에요.

- **블라인드 비교**: 어느 모델인지 숨기고 선택
- **공개 비교**: 모델명 노출하고 비교
- **AI 자동 평가**: 별도 AI가 채점해서 점수화

교재 제작 전에 "이 주제엔 어느 모델이 잘 맞을까" 테스트해보시기 좋아요.

## 자주 묻는 질문

**Q. 교재 한 권 만드는 데 얼마나 걸리나요?**
A. 10챕터 기준, 배치 생성으로 하시면 **30분~1시간** 정도. 이미지까지 넣고 검토하면 반나절 정도 보시면 돼요.

**Q. 비용은 얼마나 나와요?**
A. Sonnet 4.6 기준 **교재 한 권당 약 $1~5**. Haiku로 초안, 중요 챕터만 Opus로 다시 뽑으시면 더 저렴합니다. (관리자가 키를 제공하는 공용 서버에서는 선생님 비용 부담 없음)

**Q. 만든 교재를 다른 선생님과 공유할 수 있나요?**
A. 네, 두 가지 방법이에요. ① GitHub Pages로 배포한 URL 공유, ② Word 파일로 다운받아 전달. 같은 서버 안에서는 포트폴리오가 공개돼서 서로 참고 가능해요.

**Q. 생성된 내용이 마음에 안 들면요?**
A. 언제든 재생성·수정·삭제 가능해요. 모든 단계에서 선생님이 최종 결정자예요.

**Q. 수정한 게 학생한테 바로 반영되나요?**
A. 웹으로 배포하신 경우 **다시 배포하셔야** 반영돼요. 배포 버튼 한 번 누르시면 끝.

**Q. 작업 중인 교재가 날아가면 어떡하죠?**
A. 모든 변경은 자동 저장돼요. 브라우저를 닫으셔도, 로그아웃하셨다가 다시 오셔도 이어서 작업하실 수 있어요.

## 문제 해결

| 증상 | 해결 |
|---|---|
| "API 키가 필요합니다" | 좌측 사이드바 API 키 버튼으로 입력 (공용 서버라면 관리자에게 문의) |
| 챕터 생성이 중간에 멈춰요 | AI 프로바이더 Rate Limit 가능성. 잠시 후 재시도 또는 동시 실행 수를 줄여보세요 |
| 이미지가 SVG로만 나와요 | Gemini/OpenAI 이미지 API 키가 없을 때. 관리자에 문의하거나 키 추가 |
| 목차가 너무 많이 생성됐어요 | 방향성 논의나 피드백에서 "N차시로 맞춰주세요"라고 요청 |
| 배포한 사이트가 404 | GitHub Pages 활성화에 몇 분 걸려요. 5분 후 재시도 |
| 이전에 만든 교재가 안 보여요 | 로그인 계정이 다른 건 아닌지 확인 |
